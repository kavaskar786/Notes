# ðŸ”¥ **Vitest Matchers: Detailed Explanation**

Vitest provides **matchers** to verify test expectations. The `expect()` function is used with these matchers to compare actual vs. expected values.

---

## ðŸŸ¢ **1. Basic Equality Matchers**

These check if values are strictly or deeply equal.

### âœ… `toBe()`: Strict Equality (like `===`)

- **Compares values and types**.
- **Does not work** for objects or arrays (checks reference, not content).

#### **Scenario:** Checking primitive values

```ts
import { expect, test } from 'vitest'

test('strict equality', () => {
  expect(5).toBe(5)  // âœ… Passes
  expect('hello').toBe('hello')  // âœ… Passes
  expect(5).toBe('5')  // âŒ Fails (number !== string)
})
```

---

### âœ… `toEqual()`: Deep Equality

- **Works for objects, arrays, and nested structures**.
- Compares **content, not reference**.

#### **Scenario:** Comparing objects and arrays

```ts
test('deep equality', () => {
  expect({ a: 1 }).toEqual({ a: 1 })  // âœ… Passes
  expect([1, 2, 3]).toEqual([1, 2, 3])  // âœ… Passes
  expect({ a: 1 }).toBe({ a: 1 })  // âŒ Fails (different object references)
})
```

---

### âœ… `toStrictEqual()`: Deep Equality + Exact Type Checking

- Like `toEqual()`, but **checks undefined properties**.

#### **Scenario:** Object comparison with extra properties

```ts
test('strict deep equality', () => {
  expect({ a: 1, b: undefined }).toEqual({ a: 1 })  // âœ… Passes
  expect({ a: 1, b: undefined }).toStrictEqual({ a: 1 })  // âŒ Fails (b is missing)
})
```

---

## ðŸŸ¢ **2. Truthiness Matchers**

These check if a value is truthy, falsy, null, or undefined.

### âœ… `toBeTruthy()`: Checks if a value is **truthy**

- Any **non-falsy** value passes.

#### **Scenario:** Checking if a function returns a valid value

```ts
test('truthy values', () => {
  expect(1).toBeTruthy()  // âœ… Passes
  expect('hello').toBeTruthy()  // âœ… Passes
  expect([]).toBeTruthy()  // âœ… Passes
  expect(0).toBeTruthy()  // âŒ Fails (0 is falsy)
})
```

---

### âœ… `toBeFalsy()`: Checks if a value is **falsy**

- **Falsy values:** `false`, `0`, `''`, `null`, `undefined`, `NaN`.

#### **Scenario:** Checking invalid user input

```ts
test('falsy values', () => {
  expect('').toBeFalsy()  // âœ… Passes
  expect(0).toBeFalsy()  // âœ… Passes
  expect(null).toBeFalsy()  // âœ… Passes
  expect('hello').toBeFalsy()  // âŒ Fails ('hello' is truthy)
})
```

---

### âœ… `toBeNull()`: Checks if a value is exactly `null`

```ts
test('null check', () => {
  expect(null).toBeNull()  // âœ… Passes
  expect(undefined).toBeNull()  // âŒ Fails
})
```

---

### âœ… `toBeUndefined()`: Checks if a value is `undefined`

```ts
test('undefined check', () => {
  expect(undefined).toBeUndefined()  // âœ… Passes
  expect(null).toBeUndefined()  // âŒ Fails
})
```

## ðŸŸ¢ **3. Number Matchers**

These check numeric comparisons.

### âœ… `toBeGreaterThan()` / `toBeGreaterThanOrEqual()`

```ts
  expect(10).toBeGreaterThan(5)  // âœ… Passes
  expect(10).toBeGreaterThanOrEqual(10)  // âœ… Passes

```

---

### âœ… `toBeLessThan()` / `toBeLessThanOrEqual()`

```ts

  expect(3).toBeLessThan(5)  // âœ… Passes
  expect(5).toBeLessThanOrEqual(5)  // âœ… Passes
})
```

---

### âœ… `toBeCloseTo()`: Checks floating-point precision

```ts

})
```

## ðŸŸ¢ **4. String & Array Matchers**

### âœ… `toContain()`: Checks if an array or string contains a value


### âœ… `toHaveLength()`: Checks length of arrays or strings

---

## ðŸŸ¢ **5. Object Property Matchers**

### âœ… `toHaveProperty()`: Checks if an object has a property

## ðŸŸ¢ **6. Exception Handling**

### âœ… `toThrow()`: Checks if a function throws an error
